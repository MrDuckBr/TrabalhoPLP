Paradigma Funcional

O paradigma Funcional é um paradigma de programação,que trata a computação como uma avaliação de funções matemáticas e que evita estados ou dados mutáveis.
O paradigma funcional é um paradigma de programação assim como o orientado a objetos(POO) e o imperativo (existem outros paradigmas).
Entende-se como paradigma uma forma de fazer algo. Ou seja, paradigma de programação é o nome que se dá a maneira como se programa, a orientação que seus códigos irão ter.

O paradigma funcional eu não dito ao meu código o que ele deve fazer, quando e como. Não irei desenvolvê-lo passo a passo. Eu penso meu código como uma sequência de funções e/ou passos, as quais de maneira composta irão resolver meu problema.
De maneira simples: código funcional é um código composto de múltiplas funções que se compõem para resolver um problema

Imutabilidade: Não use variáveis ,mas sim use constantes.

Abstração: Implementação de uma função para um determinado bloco de código a ser utilizado e reutilizado.

Efeito Colateral: tendo constantes que não podem mudar,não ocorrera de ter efeitos colaterais,pelo simples fato que o valor delas não muda.

Currying:funções que recebem mais de um parâmetro que, internamente, ela será decomposta para múltiplas funções aplicando-se a técnica de currying.

Funções, funções, funções...


Não é a toa que programação funcional é a programação “orientada a funções”. O conceito de função matemática permeia este paradigma.
Funções puras: Recebem ao menos um parâmetro e trabalha com ele ,que sempre retorna algo.
